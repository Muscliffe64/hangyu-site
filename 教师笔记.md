# 教师笔记 · 学生痛点收集

> 这份文档不参与网站部署，是给 Hangyu 自己存的"学生会卡在哪"的实地观察。等走完整个流程，整理成 3 节课的教师讲义。

---

## #1 — "我不知道我有什么作品"
**场景**：开课第一节让学生开始建网站，发现他们坐在那儿想不起自己有什么。
**对策**：开课前一周发素材清单（名字 + 一句话身份 + 想展示的板块 + 各板块的几件代表作），让他们用一周时间收集。**不是作业**，相当于"上烹饪课前自己买好菜"。

## #2 — "网站结构想不清楚，本能反应是全堆首页"
**场景**：让学生设计网站结构，他们容易把所有内容塞首页。
**对策**：教一个简单决策方法论：
- 内容多就分页、内容少就单页
- 图为主用网格 / 瀑布流；文为主用列表
- 内容会持续更新就用外部平台或博客系统；不会就静态文章页

## #3 — "微信公众号导出的图片在网站上加载不出来"
**场景**：学生从公众号导出 .md 文件后，里面的封面图链接（mmbiz.qpic.cn）在自己网站上 404。
**原因**：微信对图床做了"防盗链"，外站访问会被拒绝。
**对策**：教学生"图片来源决策树"——
- 自己拍的 → 放本地 assets 文件夹
- 公众号 / 小红书导出的 → 不能直接用，要先下载存本地或重新上传到允许外链的图床（比如 Vercel、Cloudinary）
- 网络图 → 留意版权 + 防盗链

## #4 — "Cowork / Claude Design / Kimi 网页生成 哪个工具好？"
**场景**：学生面对多个 AI coding 工具时困惑，搞不清楚区别。
**核心区分**：A 类（浏览器实时预览：Claude artifact / Kimi 网页生成 / v0.dev）vs B 类（硬盘建文件：Cowork / Kimi CLI / Codex CLI / Cursor）
- A 类擅长单页面 + 视觉探索 + 零门槛
- B 类擅长多页面 + 复杂结构 + 部署上线
**对策**：第一节课开头讲清楚分叉。然后告诉学生：
- 简单单页作品集 → 用 A 类够了
- 多页面、有博客、想要"自己的文件" → 用 B 类
**比喻**：A 类是设计稿，B 类是施工。两者也可以配合用——先在 A 类探索风格，再让 B 类在自己文件里实现。

## #5 — "AI 默认会给你一个'巨大字标 hero'，但这对摄影/影像作品集是错的"
**场景**：让 AI 生成作品集首页，AI 几乎一定给你一个屏幕高度的巨型英文 wordmark（因为训练数据里太多设计师作品集模板）。
**问题**：对摄影师/纪录片人/视频创作者来说，**作品本身才是主角，文字应该退到注释位**。巨大 wordmark 反而在和自己的作品抢戏。
**对策**：教学生第一节课开头讲一条原则：
> **网站第一屏放什么，应该取决于你的作品是什么。**
> · 摄影 / 影像 / 视频 / 产品摄影为主 → 第一屏放作品（全屏图、画廊、视频）
> · 写作 / 文学 / 评论为主 → 第一屏可以放标题 + 一段引文
> · 字体 / 平面 / 视觉设计为主 → 第一屏可以是 typography hero（因为字体本身就是作品）
**Prompt 矫正词**：让学生在跟 AI 说"做一个作品集"时加一句"first screen should be image-led, not typography-led, treat the photograph as the hero"。

## #6 — "我有几百张照片，到底该怎么排？"
**场景**：学生（创作者）有大量照片要做摄影板块，纠结于排序。
**问题**：试图给"无明显关联的 100+ 张照片"排出一个总体叙事顺序，是吃力不讨好。
**对策（4 条）**：
1. **先做减法**：100 张里精选 30-50 张能站得住的，剩下不上网。**作品集是精选，不是档案库。**
2. **板块内不强求顺序**：masonry / 瀑布流让照片按视觉节奏自动排，不按故事线排
3. **靠 caption 给信息**：每张点开看大图时显示 地点 + 年份 + 一句小注 —— **单张自带上下文**
4. **EXIF 自动排时间** 作为默认（最近在前），免得手排
**附加情况**："长时段单一对象记录"是个例外——这种类型必须按时间纵深，垂直时间线滚动，年份当锚点。摄影史上是独立的一种类型（Sally Mann、Nicholas Nixon "Brown Sisters"）。Hangyu 的"积木"板块就属于这一类。

## #7 — "我想让网站每天变化（比如每天换一组照片），是不是必须有服务器？"
**场景**：学生想做"每天选一组不同照片当首页"这种动态效果，第一反应以为得有后端服务器、数据库、定时任务。
**真相**：纯静态网站也能做"看起来每天变化"的效果。**关键技巧：用今天的日期当随机种子（seed），做伪随机洗牌。**
- 同一天，所有访客的浏览器都会用同一个种子（比如 20260506）→ 算出同一个洗牌顺序 → 看到的"今日选辑"完全一致
- 跨到第二天，种子变成 20260507 → 完全不同的顺序
- 全部在浏览器本地运行，零服务器开销
**这个网站的实现** 在 `script.js` 里有完整注释代码，可以当教学样例。
**Prompt 模板**：让学生跟 AI 说"用今天的日期当 seed 做 pseudo-random shuffle，让 picks 在同一天保持稳定，跨天变化"。
**延伸**：这个思路可以做"今日推荐文章 / 今日金句 / 今日壁纸"等等——任何"每天变一次但同一天稳定"的效果都用这一招。

## #8 — "我从其他平台导出的 markdown，里面图片链接能直接用吗？"
**场景**：学生从公众号 / Portfoliobox / 小红书 / Notion / 飞书等平台导出 .md，里面的图片是平台 CDN 链接。
**真相 — 看 URL 域名分情况**：
- `mmbiz.qpic.cn`（微信公众号）→ **不能直接用**，有防盗链，外站访问会 403
- `dkemhji6i1k0x.cloudfront.net` 类（Portfoliobox / 大部分付费图床）→ **能直接用**，但依赖你的原平台账号活着
- `secure.notion-static.com`（Notion）→ **链接会过期**（一段时间后失效）
- `sns-img-bd.xhscdn.com`（小红书）→ **不能直接用**，有防盗链
- 自己服务器 / 自己 OSS → **完全可控**
**对策**：
1. **短期**：能直接用的就直接用（省事），代码里只放 URL
2. **长期 / 真心想拥有自己网站的**：把图下载到 `assets/` 文件夹，`<img src>` 改成本地路径。这样不依赖任何外部平台，也方便部署
**Hangyu 的"积木成长记"** 现在用的是 Portfoliobox CloudFront URL，能跑通，但等 Portfoliobox 不续费图就坏。下载本地是个跨进度的大动作（几百 MB 起步），等其他都稳定了再做。
**Prompt 模板**：让学生跟 AI 说"identify which image URLs in this markdown are hotlink-protected vs safely usable from a different domain"，AI 能看域名给出判断。

## #9 — "AI 默认给我加的'高级感纹理/复古滤镜'反而把我的照片搞脏了"
**场景**：让 AI 做"复古"风格的网站，AI 几乎一定给你加：film grain（胶片颗粒）、米白/牛皮纸底、做旧滤镜、纸张纹理。这些独立看着挺美。但当你的网站是图像主导（摄影 / 影像 / 视频），且使用了 `mix-blend-mode: multiply` 之类的全屏覆盖时，**这些纹理会嵌入到照片里，让你的真实作品看起来像加了一层旧滤镜**。
**核心原则**：
> **图像主导的网站，背景必须让步给作品。**
> "复古感"应该通过 typography（衬线字、斜体英文小字）、克制的细线、留白、版心比例来传达——**不靠背景纹理**。

**对比参考**：
- 错误样板：背景米白 + 全屏胶片颗粒 + 复古字体 → 颗粒污染照片
- 正确样板：纯白背景 + 衬线字体 + 0 纹理 → 美术馆白墙的气质（杉本博司、Magnum、Annie Leibovitz 的官网都是这个路子）

**对策（给学生）**：教一个判断方法："你的网站第一屏放图还是放字？放图 → 不能给整页加 texture overlay；放字 → 可以加纹理增加质感。"
**Prompt 矫正词**：在跟 AI 说"复古风"时加一句"but no full-page grain or texture overlay—keep backgrounds clean to let the imagery breathe"。

## #10 — "AI 默认会把所有照片裁成正方形，但这丢失了照片本身的语言"
**场景**：AI 做摄影 / 影像 / 作品集网站时，**默认会用 1:1 方格 grid**（来自 Instagram profile 风格的模板影响）。
**问题**：1:1 强制裁切会**丢失照片原本的构图意图**——竖构图人像被砍腿、横构图风光被砍掉左右、全景照片完全废了。摄影师的"取景"本身就是作品的一部分，强制裁切是对作品的破坏。
**4 种主流摄影画廊布局** — 各有适用场景：
| 布局 | 优点 | 缺点 | 适合 |
|---|---|---|---|
| 1:1 强制方格（Instagram profile 风格） | 节奏一致、整齐 | 暴力裁切、丢构图 | 形象品牌一致性 > 内容（个人 IG profile） |
| 自适应行（Flickr、Google Photos） | 完全保留比例、不裁剪 | 行高每行不同，需要 JS 计算 | 摄影记者、新闻图、画廊 |
| 马赛克 + 行/列跨度（这个网站采用的） | 保留大致比例 + 视觉节奏 | 仍有少量裁切（10-15%） | 生活作品集、旅行摄影 |
| Pinterest 列优先瀑布流 | 完全保留比例 | 阅读顺序是列优先（不直觉） | 灵感板、独立卡片 |
**核心原则**：**让画廊布局服从内容性质**——
- 内容是"瞬间集合"（生活、旅行）→ 马赛克跨度最自然
- 内容是"作品集"（每张都重要）→ 自适应行
- 内容是"系列"（同主题多变体）→ 1:1 强制方格 OK（因为构图相似）

**Prompt 矫正词**：让学生跟 AI 说"don't force square crops—use a mosaic grid where portrait photos span 2 rows, landscape photos span 2 columns, with auto-detection of orientation by aspect ratio"。

## #11 — "纯 flex 自然换行" vs "Flickr 自适应行" — 上百张照片画廊的两种解法
**场景**：作品集要展示 30+ 张照片，纵横比各异。
**学生第一反应（错误直觉）**："照片排不齐是因为照片不够，再加几张就好了。" **不是**——是 flex-wrap 算法本身就这样：每行装到下一张装不下就换行，行尾必然有空隙。
**两种正解**：

| 方法 | 原理 | 优 | 缺 |
|---|---|---|---|
| **纯 CSS flex 换行** | photo 高度固定，width = aspect × height，flex-wrap 自动 | 纯 CSS、零 JS | 行尾参差、看着像内容不够 |
| **Flickr 自适应行（justified rows）** | 每行单独算高度，让整行铺满容器：`h = (容器宽 - 间隔) / Σaspect` | 行尾整齐、视觉精炼 | 需要 JS 等所有图加载完算 |

**哪个时候选哪个**：
- 30 张以下、追求极简感 → 纯 flex 够了
- 30 张以上、追求专业作品集感 → 必须 justified rows
- Hangyu 的"积木成长记"131 张 + 首页选辑都用了 justified

**Prompt 模板**：让学生跟 AI 说 "implement Flickr-style justified rows layout: each row's height is computed so the row exactly fills the container width, photos preserve aspect ratios with no cropping"。

## #12 — Lightbox 是值得专门讲的"标准 UI 模式"
**场景**：摄影/影像作品集 — 访客想看大图但不想离开当前页。
**核心模式**：点击缩略图 → 浮层弹出大图 + 背景模糊 + 点击空白或 ESC 关闭。
**为什么这是"该教的标准动作"**：
- 几乎所有像样的摄影/设计作品集都有，几乎是 user expectation
- 实现成本极低：~50 行 JS + ~30 行 CSS
- 鼓励"沉浸式浏览"，访客滞留时间显著延长
**关键技术点（讲给学生）**：
- **事件代理**：在 `document` 上监听 click，用 `e.target.closest('.photo')` 判断是不是照片点击。这样**新加的照片自动也响应**，不用每张照片单独绑事件
- **`backdrop-filter: blur(20px)`**：CSS 一行实现毛玻璃效果（注意 iOS 要 `-webkit-backdrop-filter`）
- **`visibility + opacity` 组合**：实现平滑显隐过渡（单独 display:none 不能动画）
- **`document.body.style.overflow = 'hidden'`**：弹出时锁住背景滚动
**Prompt 模板**：让学生跟 AI 说 "add a lightbox: clicking any image opens it in a fullscreen modal with blurred backdrop, click outside the image or press ESC to close"。
